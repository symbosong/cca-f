## 第一部分：工具使用（Tool Use / Function Calling）

### 第 1 课：工具使用概览

> 📓 `tool_use/01_tool_use_overview.ipynb`

#### 什么是工具使用？

工具使用（Tool Use），也叫函数调用（Function Calling），是让 Claude 调用外部函数和 API 的能力。

**关键理解**：Claude 本身**不执行**工具——它只是告诉你**应该调用哪个工具、传什么参数**。实际执行由你的代码完成。

```
用户请求 → Claude 分析 → Claude 决定使用工具 → 你的代码执行工具 → 结果返回给 Claude → Claude 生成最终回复
```

#### 典型场景

| 场景 | 工具示例 |
|------|---------|
| 查询实时信息 | 天气 API、股票 API、搜索引擎 |
| 数据库操作 | 查询用户信息、订单状态 |
| 精确计算 | 数学计算器、单位转换 |
| 系统集成 | 发邮件、创建日历事件 |
| 文件操作 | 读写文件、处理数据 |

#### 工具使用的完整流程

```
步骤 1：你定义工具（JSON Schema）并随请求发送
步骤 2：Claude 决定是否需要使用工具
步骤 3：如果需要，Claude 返回 stop_reason="tool_use" 和工具调用信息
步骤 4：你的代码执行工具，获取结果
步骤 5：将结果以 tool_result 消息返回给 Claude
步骤 6：Claude 基于工具结果生成最终回复
（如果 Claude 需要更多工具调用，重复步骤 2-6）
```

### 第 2 课：定义和使用工具

> 📓 `tool_use/02_your_first_simple_tool.ipynb`

#### 定义工具的 JSON Schema

每个工具需要：
- `name`：工具名（Claude 用来标识）
- `description`：描述工具做什么（**非常重要**——Claude 依赖描述来决定何时使用）
- `input_schema`：输入参数的 JSON Schema

```python
import anthropic

client = anthropic.Anthropic()

# 定义一个计算器工具
tools = [
    {
        "name": "calculator",
        "description": "A simple calculator that performs basic arithmetic operations. Use this tool whenever you need to perform mathematical calculations to ensure accuracy.",
        "input_schema": {
            "type": "object",
            "properties": {
                "expression": {
                    "type": "string",
                    "description": "The mathematical expression to evaluate, e.g., '2 + 3 * 4'"
                }
            },
            "required": ["expression"]
        }
    }
]

# 发送带工具的请求
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=tools,
    messages=[
        {"role": "user", "content": "What is 1984135 * 9343116?"}
    ]
)

print(response.stop_reason)  # "tool_use"
print(response.content)
```

#### 解析工具调用

```python
# Claude 的响应 content 是一个列表，可能包含文本和工具调用
for block in response.content:
    if block.type == "tool_use":
        print(f"工具名: {block.name}")
        print(f"工具ID: {block.id}")
        print(f"输入参数: {block.input}")
        # 输出示例:
        # 工具名: calculator
        # 工具ID: toolu_01ABC...
        # 输入参数: {"expression": "1984135 * 9343116"}
    elif block.type == "text":
        print(f"文本: {block.text}")
```

#### 完成工具调用循环

```python
def process_tool_call(tool_name, tool_input):
    """执行工具并返回结果"""
    if tool_name == "calculator":
        try:
            result = eval(tool_input["expression"])
            return str(result)
        except Exception as e:
            return f"Error: {str(e)}"

# 1. 发送初始请求
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=tools,
    messages=[
        {"role": "user", "content": "What is 1984135 * 9343116?"}
    ]
)

# 2. 检查是否需要工具调用
if response.stop_reason == "tool_use":
    # 找到工具调用块
    tool_use_block = next(
        block for block in response.content if block.type == "tool_use"
    )
    
    # 3. 执行工具
    tool_result = process_tool_call(
        tool_use_block.name, 
        tool_use_block.input
    )
    
    # 4. 将工具结果返回给 Claude
    follow_up = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        tools=tools,
        messages=[
            {"role": "user", "content": "What is 1984135 * 9343116?"},
            {"role": "assistant", "content": response.content},  # Claude 之前的回复
            {
                "role": "user",
                "content": [
                    {
                        "type": "tool_result",
                        "tool_use_id": tool_use_block.id,
                        "content": tool_result,
                    }
                ]
            }
        ]
    )
    
    # 5. 获取最终回复
    print(follow_up.content[0].text)
```

#### 通用工具调用循环（Agentic Loop）

这是最重要的模式——**循环处理所有工具调用直到 Claude 完成**：

```python
def run_tool_loop(user_message, tools, max_iterations=10):
    """
    通用工具调用循环。
    持续处理工具调用直到 Claude 给出最终回复。
    """
    messages = [{"role": "user", "content": user_message}]
    
    for i in range(max_iterations):
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=4096,
            tools=tools,
            messages=messages
        )
        
        # 如果 Claude 不需要工具调用，返回最终回复
        if response.stop_reason == "end_turn":
            final_text = ""
            for block in response.content:
                if hasattr(block, "text"):
                    final_text += block.text
            return final_text
        
        # 处理工具调用
        if response.stop_reason == "tool_use":
            # 将 Claude 的响应加入消息历史
            messages.append({"role": "assistant", "content": response.content})
            
            # 执行所有工具调用
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    print(f"  🔧 调用工具: {block.name}({json.dumps(block.input)})")
                    result = process_tool_call(block.name, block.input)
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": str(result),
                    })
            
            # 将工具结果发回给 Claude
            messages.append({"role": "user", "content": tool_results})
    
    return "达到最大迭代次数"

# 使用示例
result = run_tool_loop(
    "What's the weather in Tokyo and what time is it there?",
    tools
)
print(result)
```

#### tool_choice 参数

你可以控制 Claude 如何使用工具：

```python
# 方式 1：自动（默认）— Claude 自己决定是否用工具
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=tools,
    tool_choice={"type": "auto"},  # 默认行为
    messages=[{"role": "user", "content": "Hello!"}]
)

# 方式 2：强制使用特定工具 — Claude 必须使用该工具
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=tools,
    tool_choice={"type": "tool", "name": "get_weather"},  # 强制使用 get_weather
    messages=[{"role": "user", "content": "Tell me about Paris"}]
)

# 方式 3：强制使用任意工具 — Claude 必须至少使用一个工具
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=tools,
    tool_choice={"type": "any"},  # 必须用一个工具
    messages=[{"role": "user", "content": "Hello!"}]
)

# 方式 4: 禁用工具 — 即使定义了工具，也不会使用
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=tools,
    tool_choice={"type": "none"},  # 不使用工具
    messages=[{"role": "user", "content": "What's 2+2?"}]
)
```

```python
        return [o for o in self.orders if o["customer_id"] == customer_id]

        """取消订单"""
        order = self.get_order_by_id(order_id)
        if order:
            if order["status"] == "processing":
                order["status"] = "cancelled"

            else:
```

## 第二部分：高级特性

### Extended Thinking（扩展思考）

> 📹 对应视频：Extended Thinking 课程

#### 什么是 Extended Thinking？

Extended Thinking 让 Claude 在回答之前进行深度"思考"——类似于人类面对困难问题时会先在脑中推理一遍。

```python
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=16000,
    thinking={
        "type": "enabled",
        "budget_tokens": 10000  # 给思考过程分配的最大 token 数
    },
    messages=[{
        "role": "user",
        "content": "Solve this step by step: If a train leaves Station A at 9:00 AM traveling at 60 mph, and another train leaves Station B (300 miles away) at 10:00 AM traveling at 90 mph toward Station A, at what time do they meet?"
    }]
)

# 响应包含思考块和回答块
for block in response.content:
    if block.type == "thinking":
        print("🧠 思考过程:")
        print(block.thinking)
        print()
    elif block.type == "text":
        print("💬 最终回答:")
        print(block.text)
```

#### 关键参数

| 参数 | 说明 |
|------|------|
| `thinking.type` | `"enabled"` 启用扩展思考 |
| `thinking.budget_tokens` | 思考的最大 token 预算（越大思考越深入，但更慢更贵） |

#### 何时使用 Extended Thinking

| ✅ 适合 | ❌ 不适合 |
|---------|---------|
| 数学/逻辑推理 | 简单问答 |
| 复杂代码生成 | 分类/提取 |
| 多步骤分析 | 简单翻译 |
| 策略规划 | 格式化/改写 |

#### 注意事项

- Extended Thinking 会消耗额外的 token（输入 + 思考 + 输出）
- 使用扩展思考时 **不能设置 `temperature`**（固定为 1.0）
- 使用扩展思考时 **不能使用 `system` prompt**（限制已放宽，请查看最新文档）
- `budget_tokens` 不是保证用完那么多 token，而是最大限制

### Prompt Caching（提示缓存）

> 📹 对应视频：Prompt Caching 课程

#### 什么是 Prompt Caching？

当你反复发送包含大量相同内容的请求时（如系统提示、工具定义、长文档），Prompt Caching 可以缓存这些静态部分，**大幅降低成本和延迟**。

```python
# 不使用缓存 — 每次都发送完整的系统提示
# 如果系统提示有 10000 tokens，每次请求都要付 10000 个 input token

# 使用缓存 — 在需要缓存的内容上添加 cache_control
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    system=[
        {
            "type": "text",
            "text": "这里是一个很长的系统提示..." * 100,  # 假设很长
            "cache_control": {"type": "ephemeral"}  # 标记为可缓存
        }
    ],
    messages=[{"role": "user", "content": "Hello!"}]
)

# 查看缓存效果
print(f"输入 tokens: {response.usage.input_tokens}")
print(f"缓存创建 tokens: {response.usage.cache_creation_input_tokens}")  # 首次创建缓存
print(f"缓存命中 tokens: {response.usage.cache_read_input_tokens}")     # 后续命中缓存
```

#### 缓存使用规则

1. **最小缓存大小**：缓存内容必须超过 1024 tokens（Haiku 为 2048）
2. **缓存有效期**：5 分钟（ephemeral），在有效期内重复请求会命中缓存
3. **可缓存内容**：system prompt、tools 定义、messages 中的前缀部分
4. **费用节省**：缓存命中的 token 费用仅为正常输入的 **10%**

#### 缓存最佳实践

```python
# 将静态内容（工具定义、系统提示、参考文档）放在消息前面并标记缓存
# 将动态内容（用户消息）放在最后

long_document = "..." * 5000  # 假设是一份很长的参考文档

# 第一次请求（创建缓存）
```

### Batch API（批处理 API）

```python
# 批处理 — 成本降低 50%
import anthropic

        },
        {
```

## 工具使用最佳实践

### 1. 工具描述是关键

```python
# ❌ 差的描述
{
    "name": "search",
    "description": "Search for things"  # 太模糊
}

# ✅ 好的描述
{
    "name": "search_products",
    "description": "Search the product catalog by keyword or category. Returns a list of matching products with name, price, and availability. Use this when the customer asks about products, pricing, or availability. Do NOT use this for order-related queries."
}
```

### 2. 合理设计工具数量

- **< 5 个工具**：效果最好
- **5-20 个工具**：需要非常好的描述
- **> 20 个工具**：考虑分组或使用二级工具选择

### 3. 错误处理

```python
def process_tool_call(tool_name, tool_input):
    """工具调用要有错误处理"""
    try:
        if tool_name == "get_weather":
            result = call_weather_api(tool_input["location"])
            return json.dumps(result)
    except Exception as e:
        # 返回有意义的错误信息，让 Claude 可以优雅地处理
        return json.dumps({
            "error": True,
            "message": f"Failed to get weather: {str(e)}"
        })
```
