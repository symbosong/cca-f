## 第 1 课：工具使用概览

> 📹 视频：Introducing Tool Use（3 min）  
> 📓 配套 Notebook：`01_tool_use_overview.ipynb`

### 什么是工具使用？

工具使用（Tool Use），也叫函数调用（Function Calling），是让 Claude 调用外部函数和 API 的能力。

**关键理解**：Claude 本身**不执行**工具——它只是告诉你**应该调用哪个工具、传什么参数**。实际执行由你的代码完成。

```
用户请求 → Claude 分析 → Claude 决定使用工具 → 你的代码执行工具 → 结果返回给 Claude → Claude 生成最终回复
```

### 典型场景

| 场景 | 工具示例 |
|------|---------|
| 查询实时信息 | 天气 API、股票 API、搜索引擎 |
| 数据库操作 | 查询用户信息、订单状态 |
| 精确计算 | 数学计算器、单位转换 |
| 系统集成 | 发邮件、创建日历事件 |
| 文件操作 | 读写文件、处理数据 |

### 工具使用的完整流程

```
步骤 1：你定义工具（JSON Schema）并随请求发送
步骤 2：Claude 决定是否需要使用工具
步骤 3：如果需要，Claude 返回 stop_reason="tool_use" 和工具调用信息
步骤 4：你的代码执行工具，获取结果
步骤 5：将结果以 tool_result 消息返回给 Claude
步骤 6：Claude 基于工具结果生成最终回复
（如果 Claude 需要更多工具调用，重复步骤 2-6）
```

### 工具函数是你自己的代码

工具函数就是普通的 Python 函数——Claude 会告诉你调用哪个函数、传什么参数，但执行是你的事：

```python
def process_tool_call(tool_name, tool_input):
    """执行工具并返回结果"""
    if tool_name == "calculator":
        try:
            result = eval(tool_input["expression"])
            return str(result)
        except Exception as e:
            return f"Error: {str(e)}"
```

## 第 4 课：定义工具的 JSON Schema

> 📹 视频：Tool Schemas（5 min）  
> 📓 配套 Notebook：`02_your_first_simple_tool.ipynb`

### description 是 Claude 选择工具的主要依据

每个工具需要三个要素：
- `name`：工具名（Claude 用来标识）
- `description`：描述工具做什么（**非常重要**——Claude 依赖描述来决定何时使用）
- `input_schema`：输入参数的 JSON Schema

```python
import anthropic

client = anthropic.Anthropic()

# 定义一个计算器工具
tools = [
    {
        "name": "calculator",
        "description": "A simple calculator that performs basic arithmetic operations. Use this tool whenever you need to perform mathematical calculations to ensure accuracy.",
        "input_schema": {
            "type": "object",
            "properties": {
                "expression": {
                    "type": "string",
                    "description": "The mathematical expression to evaluate, e.g., '2 + 3 * 4'"
                }
            },
            "required": ["expression"]
        }
    }
]

# 发送带工具的请求
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=tools,
    messages=[
        {"role": "user", "content": "What is 1984135 * 9343116?"}
    ]
)

print(response.stop_reason)  # "tool_use"
print(response.content)
```

## 第 6 课：返回工具结果

> 📹 视频：Sending Tool Results（9 min）  
> 📓 配套 Notebook：`04_complete_workflow.ipynb`

### tool_result 消息格式

将工具执行结果以 `tool_result` 消息返回给 Claude，注意 `tool_use_id` 必须匹配：

```python
# 1. 发送初始请求
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=tools,
    messages=[
        {"role": "user", "content": "What is 1984135 * 9343116?"}
    ]
)

# 2. 找到工具调用块
if response.stop_reason == "tool_use":
    tool_use_block = next(
        block for block in response.content if block.type == "tool_use"
    )
    
    # 3. 执行工具
    tool_result = process_tool_call(
        tool_use_block.name, 
        tool_use_block.input
    )
    
    # 4. 将工具结果返回给 Claude
    follow_up = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        tools=tools,
        messages=[
            {"role": "user", "content": "What is 1984135 * 9343116?"},
            {"role": "assistant", "content": response.content},
            {
                "role": "user",
                "content": [
                    {
                        "type": "tool_result",
                        "tool_use_id": tool_use_block.id,
                        "content": tool_result,
                    }
                ]
            }
        ]
    )
    
    print(follow_up.content[0].text)
```

## 第 7 课：带工具的多轮对话

> 📹 视频：Multi-turn Tool Use（9 min）  
> 📓 配套 Notebook：`04_complete_workflow.ipynb`

### 通用工具调用循环（Agentic Loop）

这是最重要的模式——**循环处理所有工具调用直到 Claude 完成**：

```python
import json

def run_tool_loop(user_message, tools, max_iterations=10):
    """
    通用工具调用循环。
    持续处理工具调用直到 Claude 给出最终回复。
    """
    messages = [{"role": "user", "content": user_message}]
    
    for i in range(max_iterations):
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=4096,
            tools=tools,
            messages=messages
        )
        
        # 如果 Claude 不需要工具调用，返回最终回复
        if response.stop_reason == "end_turn":
            final_text = ""
            for block in response.content:
                if hasattr(block, "text"):
                    final_text += block.text
            return final_text
        
        # 处理工具调用
        if response.stop_reason == "tool_use":
            messages.append({"role": "assistant", "content": response.content})
            
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    print(f"  🔧 调用工具: {block.name}({json.dumps(block.input)})")
                    result = process_tool_call(block.name, block.input)
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": str(result),
                    })
            
            messages.append({"role": "user", "content": tool_results})
    
    return "达到最大迭代次数"
```

### tool_choice 强制结构化输出

一个巧妙的技巧：利用工具的 `input_schema` 来强制 Claude 输出特定的 JSON 结构：

```python
# 定义一个"假工具"来强制结构化输出
structured_output_tool = [
    {
        "name": "format_output",
        "description": "Format the analysis results into a structured output.",
        "input_schema": {
            "type": "object",
            "properties": {
                "sentiment": {
                    "type": "string",
                    "enum": ["positive", "negative", "neutral"]
                },
                "confidence": {
                    "type": "number",
                    "description": "Confidence score from 0 to 1"
                },
                "key_topics": {
                    "type": "array",
                    "items": {"type": "string"}
                },
                "summary": {
                    "type": "string"
                }
            },
            "required": ["sentiment", "confidence", "key_topics", "summary"]
        }
    }
]

# 强制使用此工具 → 得到结构化 JSON
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=structured_output_tool,
    tool_choice={"type": "tool", "name": "format_output"},
    messages=[{
        "role": "user",
        "content": "Analyze this review: 'The new iPhone is amazing! The camera quality blew my mind, though the battery could be better.'"
    }]
)

tool_use_block = next(b for b in response.content if b.type == "tool_use")
structured_data = tool_use_block.input
print(json.dumps(structured_data, indent=2))
```

## 第 15 课：Extended Thinking（扩展思考）

> 📹 视频：Extended Thinking（7 min）

### 什么是 Extended Thinking？

Extended Thinking 让 Claude 在回答之前进行深度"思考"——类似于人类面对困难问题时会先在脑中推理一遍。

```python
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=16000,
    thinking={
        "type": "enabled",
        "budget_tokens": 10000  # 给思考过程分配的最大 token 数
    },
    messages=[{
        "role": "user",
        "content": "Solve this step by step: If a train leaves Station A at 9:00 AM traveling at 60 mph, and another train leaves Station B (300 miles away) at 10:00 AM traveling at 90 mph toward Station A, at what time do they meet?"
    }]
)

# 响应包含思考块和回答块
for block in response.content:
    if block.type == "thinking":
        print("🧠 思考过程:")
        print(block.thinking)
    elif block.type == "text":
        print("💬 最终回答:")
        print(block.text)
```

### 何时使用

| ✅ 适合 | ❌ 不适合 |
|---------|---------|
| 数学/逻辑推理 | 简单问答 |
| 复杂代码生成 | 分类/提取 |
| 多步骤分析 | 简单翻译 |
| 策略规划 | 格式化/改写 |

### 注意事项

- 使用扩展思考时 **不能设置 `temperature`**（固定为 1.0）
- `budget_tokens` 是最大限制，不是保证用完
- 会消耗额外的 token（输入 + 思考 + 输出）

## 第 19 课：Prompt Caching 概念

> 📹 视频：Prompt Caching（4 min）

### 什么是 Prompt Caching？

当你反复发送包含大量相同内容的请求时（如系统提示、工具定义、长文档），Prompt Caching 可以缓存这些静态部分，**大幅降低成本和延迟**。

```python
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    system=[
        {
            "type": "text",
            "text": "这里是一个很长的系统提示..." * 100,
            "cache_control": {"type": "ephemeral"}  # 标记为可缓存
        }
    ],
    messages=[{"role": "user", "content": "Hello!"}]
)

print(f"缓存创建 tokens: {response.usage.cache_creation_input_tokens}")
print(f"缓存命中 tokens: {response.usage.cache_read_input_tokens}")
```

## 第 20 课：Prompt Caching 规则

> 📹 视频：Prompt Caching Rules（7 min）

### 缓存使用规则

1. **最小缓存大小**：缓存内容必须超过 1024 tokens（Haiku 为 2048）
2. **缓存有效期**：5 分钟（ephemeral），在有效期内重复请求会命中缓存
3. **可缓存内容**：system prompt、tools 定义、messages 中的前缀部分
4. **费用节省**：缓存命中的 token 费用仅为正常输入的 **10%**

## 工具使用最佳实践

### 1. 工具描述是关键

```python
# ❌ 差的描述
{"name": "search", "description": "Search for things"}

# ✅ 好的描述
{
    "name": "search_products",
    "description": "Search the product catalog by keyword or category. Returns a list of matching products with name, price, and availability. Use this when the customer asks about products, pricing, or availability. Do NOT use this for order-related queries."
}
```

### 2. 合理设计工具数量

- **< 5 个工具**：效果最好
- **5-20 个工具**：需要非常好的描述
- **> 20 个工具**：考虑分组或使用二级工具选择

### 3. 错误处理

```python
def process_tool_call(tool_name, tool_input):
    """工具调用要有错误处理"""
    try:
        if tool_name == "get_weather":
            result = call_weather_api(tool_input["location"])
            return json.dumps(result)
    except Exception as e:
        return json.dumps({
            "error": True,
            "message": f"Failed to get weather: {str(e)}"
        })
```
