## 第一部分：Agent SDK — 构建生产级智能体

> **官方文档**：https://platform.claude.com/docs/en/agent-sdk/overview

### 1.1 什么是 Agent SDK

Agent SDK（前身 Claude Code SDK）是 Anthropic 官方提供的**构建自主 AI 智能体**的开发工具包。它提供了与 Claude Code 完全相同的工具集、智能体循环和上下文管理能力，但以**库**的形式供你编程调用。

**关键区别**：

| 对比项 | Messages API | Agent SDK |
|:------|:------------|:---------|
| 定位 | 单次对话 / 手动编排 | 自主智能体循环 |
| 工具执行 | 你自己写工具执行逻辑 | SDK 内置工具自动执行 |
| 上下文管理 | 你自己管理消息历史 | SDK 自动管理上下文窗口 |
| 文件操作 | 需要自己实现 | 内置 Read/Edit/Write/Glob/Grep |
| 命令执行 | 需要自己实现 | 内置 Bash 工具 |
| Web 搜索 | 需要自己实现 | 内置 WebSearch/WebFetch |

### 1.3 快速上手：5 分钟构建一个 Bug 修复 Agent

**安装**：

```bash
# Python
pip install claude-agent-sdk

# TypeScript
npm install @anthropic-ai/claude-agent-sdk
```

**设置 API Key**：

```bash
export ANTHROPIC_API_KEY="your-api-key"
```

**创建一个包含 Bug 的文件 `utils.py`**：

```python
def calculate_average(numbers):
    total = 0
    for num in numbers:
        total += num
    return total / len(numbers)  # Bug: 空列表会除零

def get_user_name(user):
    return user["name"].upper()  # Bug: user 为 None 会崩溃
```

**构建 Agent (`agent.py`)**：

```python
import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions, AssistantMessage, ResultMessage


async def main():
    # 核心：query() 返回异步迭代器，流式输出 Agent 的每一步动作
    async for message in query(
        prompt="Review utils.py for bugs that would cause crashes. Fix any issues you find.",
        options=ClaudeAgentOptions(
            allowed_tools=["Read", "Edit", "Glob"],  # 授权使用的工具
            permission_mode="acceptEdits",             # 自动批准文件编辑
        ),
    ):
        if isinstance(message, AssistantMessage):
            for block in message.content:
                if hasattr(block, "text"):
                    print(block.text)        # Claude 的推理过程
                elif hasattr(block, "name"):
                    print(f"🔧 Tool: {block.name}")  # 正在调用的工具
        elif isinstance(message, ResultMessage):
            print(f"✅ Done: {message.subtype}")


asyncio.run(main())
```

**运行**：

```bash
python agent.py
```

Agent 会自动：读取文件 → 分析 Bug → 编辑文件添加防御性代码 → 完成

### 2.1 CLAUDE.md 三级配置体系

CLAUDE.md 是 Claude Code 的「记忆」系统，分为三个层级：

```
优先级从高到低：
┌─────────────────────────────────────────┐
│  管理策略级（组织统一规则）                  │  ← IT/DevOps 管理
│  位置：/Library/Application Support/      │
│        ClaudeCode/CLAUDE.md              │
│  不可被个人覆盖                            │
├─────────────────────────────────────────┤
│  项目级（团队共享）                         │  ← 提交到 Git
│  位置：./CLAUDE.md 或                     │
│        ./.claude/CLAUDE.md               │
│  通过源代码控制共享                         │
├─────────────────────────────────────────┤
│  用户级（个人偏好）                         │  ← 仅自己
│  位置：~/.claude/CLAUDE.md               │
│  适用于所有项目                            │
└─────────────────────────────────────────┘
```

**项目级 CLAUDE.md 示例**：

```markdown
# 项目规范

## 架构
- 使用 Clean Architecture，分为 domain/application/infrastructure 三层
- API 路由在 src/routes/ 下，每个资源一个文件

## 编码标准
- TypeScript strict 模式
- 使用 Zod 做运行时类型校验
- 错误处理使用自定义 AppError 类
- 所有 public 方法必须有 JSDoc 注释

## 测试
- 单元测试使用 Vitest
- 运行测试：pnpm test
- 测试覆盖率阈值：80%

## Git 规范
- Commit message 遵循 Conventional Commits
- PR 标题格式：[类型] 简短描述

## 禁止操作
- 不要修改 .env 文件
- 不要直接操作数据库
- 不要删除 migrations 文件
```

**自动记忆（Auto Memory）**：

- Claude Code 会在工作过程中**自动保存**学到的模式和偏好
- 存储在 `~/.claude/projects/<project>/memory/MEMORY.md`
- 每次会话自动加载前 200 行
- 可用 `/memory` 命令查看和编辑

**最佳实践**：

1. 每个 CLAUDE.md 文件控制在 **200 行以内**（过长会降低遵循度）
2. 使用 Markdown 标题分组相关指令
3. 写**可验证**的具体指令（"使用 2 空格缩进" ✅ vs "正确格式化代码" ❌）
4. 大型项目用 `.claude/rules/` 目录拆分规则
5. 用 `@path/to/file` 语法引入外部文件

**路径特定规则**（仅在相关文件被读取时加载）：

```yaml
---
paths:
  - "src/api/**/*.ts"
---
# API 层规则
- 所有端点必须有速率限制
- 响应必须包含 request_id
- 错误响应遵循 RFC 7807 格式
```

### 2.2 Hooks 钩子系统

> **官方文档**：https://code.claude.com/docs/en/hooks

Hooks 是 Claude Code 生命周期中**自动执行的用户定义命令**。支持四种类型：

| 类型 | 说明 |
|:----|:-----|
| `command` | 执行 Shell 命令 |
| `http` | 发送 HTTP POST 请求到端点 |
| `prompt` | 向 Claude 发送提示词进行单轮评估 |
| `agent` | 生成一个子代理进行验证 |

#### 完整事件列表

| 事件 | 触发时机 | 可阻止？ | 典型用途 |
|:-----|:--------|:---------|:---------|
| `SessionStart` | 会话开始 | ❌ | 设置环境变量 |
| `UserPromptSubmit` | 用户提交提示词后 | ✅ | 输入校验 |
| `PreToolUse` | 工具执行前 | ✅ | 阻止危险命令 |
| `PostToolUse` | 工具执行成功后 | ❌ | 自动运行测试 |
| `Stop` | Claude 完成响应时 | ✅ | 验证任务是否真正完成 |
| `TaskCompleted` | 任务标记完成时 | ✅ | 强制测试通过 |
| `SubagentStart/Stop` | 子代理生命周期 | 部分 | 监控子代理 |
| `PreCompact/PostCompact` | 上下文压缩前后 | ❌ | 自定义压缩策略 |

#### 示例 1：阻止危险的 Shell 命令

**配置 (`.claude/settings.json`)**：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": ".claude/hooks/block-rm.sh"
          }
        ]
      }
    ]
  }
}
```

**脚本 (`.claude/hooks/block-rm.sh`)**：

```bash
#!/bin/bash
COMMAND=$(jq -r '.tool_input.command')

if echo "$COMMAND" | grep -q 'rm -rf'; then
  jq -n '{
    hookSpecificOutput: {
      hookEventName: "PreToolUse",
      permissionDecision: "deny",
      permissionDecisionReason: "rm -rf blocked by security hook"
    }
  }'
else
  exit 0
fi
```

#### 示例 2：文件编辑后自动运行测试（异步）

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "$CLAUDE_PROJECT_DIR/.claude/hooks/run-tests-async.sh",
            "async": true,
            "timeout": 300
          }
        ]
      }
    ]
  }
}
```

```bash
#!/bin/bash
INPUT=$(cat)
FILE_PATH=$(echo "$INPUT" | jq -r '.tool_input.file_path // empty')

# 仅对源文件运行测试
if [[ "$FILE_PATH" != *.ts && "$FILE_PATH" != *.js ]]; then
  exit 0
fi

RESULT=$(npm test 2>&1)
EXIT_CODE=$?

if [ $EXIT_CODE -eq 0 ]; then
  echo "{\"systemMessage\": \"Tests passed after editing $FILE_PATH\"}"
else
  echo "{\"systemMessage\": \"Tests FAILED after editing $FILE_PATH: $RESULT\"}"
fi
```

### 2.3 CI/CD 集成：GitHub Actions

> **官方文档**：https://code.claude.com/docs/en/github-actions

Claude Code 可以直接集成到 GitHub Actions 中，实现：
- **@claude 提及**：在 Issue 或 PR 评论中 @claude，它会自动分析代码并回复
- **自动代码审查**：PR 创建时自动运行 Claude 审查
- **Issue → PR**：描述需求，Claude 自动创建完整 PR

**基础工作流 (`.github/workflows/claude.yml`)**：

```yaml
name: Claude Code
on:
  issue_comment:
    types: [created]
  pull_request_review_comment:
    types: [created]

jobs:
  claude:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
```

**自动代码审查**：

```yaml
name: Code Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: >
            Review this PR for code quality, correctness, and security.
            Analyze the diff, then post findings as review comments.
          claude_args: "--max-turns 5"
```

**AWS Bedrock / Google Vertex AI 集成**：

企业场景可以不直接调用 Anthropic API，而是通过自己的云基础设施（Bedrock/Vertex）运行，实现数据驻留和计费控制。配置时使用 OIDC 进行无密钥认证：

```yaml
# AWS Bedrock 示例
- uses: anthropics/claude-code-action@v1
  with:
    github_token: ${{ steps.app-token.outputs.token }}
    use_bedrock: "true"
```

## 第三部分：Extended Thinking 深度控制

> **官方文档**：https://platform.claude.com/docs/en/build-with-claude/extended-thinking

### 3.1 核心概念

Extended Thinking 让 Claude 在给出最终答案前进行**内部推理**。启用后，响应会包含 `thinking` 块（推理过程）和 `text` 块（最终答案）。

### 3.2 两种模式

#### 手动模式（Claude Sonnet 4.6 及以下）

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=16000,
    thinking={
        "type": "enabled",
        "budget_tokens": 10000  # 思考预算：最多用 10000 token 推理
    },
    messages=[
        {"role": "user", "content": "分析这段代码的时间复杂度并给出优化建议..."}
    ]
)

# 响应结构
for block in response.content:
    if block.type == "thinking":
        print(f"💭 思考过程: {block.thinking}")
        print(f"🔒 签名: {block.signature[:50]}...")  # 加密签名，用于多轮连续性
    elif block.type == "text":
        print(f"📝 最终答案: {block.text}")
```

#### 自适应模式（Claude Opus 4.6 推荐）

```python
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=16000,
    thinking={
        "type": "adaptive",   # 自适应：Claude 自己决定思考多少
        "effort": "high"      # low / medium / high
    },
    messages=[...]
)
```

### 3.3 关键参数详解

| 参数 | 类型 | 说明 |
|:----|:-----|:-----|
| `type` | `"enabled"` / `"adaptive"` | 手动 vs 自适应模式 |
| `budget_tokens` | int | 思考预算（手动模式必填），最小 1024，必须 < max_tokens |
| `effort` | `"low"` / `"medium"` / `"high"` | 自适应模式的思考深度 |
| `display` | `"summarized"` / `"omitted"` | 控制思考内容返回方式 |

**display 参数的作用**：

- `"summarized"`（默认）：返回思考摘要。你为完整思考 token 付费，但只看到摘要。
- `"omitted"`：不返回思考文本（但 signature 仍在）。**减少延迟**，但不减少成本。

### 3.4 交错思考（Interleaved Thinking）

Claude 4 系列支持在**工具调用之间**进行思考：

```
用户提问 → 思考 → 调用工具 A → 思考（分析工具 A 结果）→ 调用工具 B → 思考 → 最终答案
```

- **Claude Opus 4.6**：使用自适应思考时自动启用
- **Claude Sonnet 4.6**：需要添加 `interleaved-thinking-2025-05-14` beta 头
- 交错思考时，`budget_tokens` 可以**超过** `max_tokens`

### 3.5 与工具使用结合的限制

1. **只能用 `tool_choice: auto`**（默认）或 `none`，不能强制指定工具
2. 必须将思考块传回 API 以保持推理连续性
3. 不能在工具使用循环中途切换思考模式

### 3.6 最佳实践

| 场景 | 推荐配置 |
|:----|:---------|
| 简单问答 | 不启用思考 |
| 代码分析 | budget_tokens: 8000-16000 |
| 复杂推理/数学 | budget_tokens: 16000+ |
| 生产环境（不需要展示思考） | display: "omitted" |
| 超长任务 | 使用 Batch API 避免网络超时 |
| Claude Opus 4.6 | 始终用 adaptive + effort |

### 3.7 定价影响

- 思考 token 按**输出 token** 计费（最贵的 token 类型）
- Opus 4.6: 思考输出 $25/MTok
- Sonnet 4.6: 思考输出 $15/MTok
- `display: "omitted"` 不减少成本，只减少延迟

## 第四部分：Prompt Caching 生产优化

> **官方文档**：https://platform.claude.com/docs/en/build-with-claude/prompt-caching

### 4.1 核心价值

Prompt Caching 允许**缓存提示词的前缀部分**，后续请求复用缓存，实现：

- **降低成本**：缓存命中只需基础价格的 **10%**
- **降低延迟**：跳过缓存部分的处理
- **提高吞吐**：缓存命中不计入速率限制

### 4.2 两种使用方式

#### 自动缓存（推荐用于多轮对话）

```python
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    cache_control={"type": "ephemeral"},  # 顶层声明，自动管理断点
    system="You are an AI assistant...",
    messages=[...]
)
```

#### 显式缓存断点（精细控制）

```python
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system=[
        {
            "type": "text",
            "text": "You are a legal document analyzer...",
        },
        {
            "type": "text",
            "text": "<entire_legal_document>...(50000 tokens)...</entire_legal_document>",
            "cache_control": {"type": "ephemeral"}  # 在此处设置缓存断点
        }
    ],
    messages=[
        {"role": "user", "content": "Summarize section 3.2"}
    ]
)
```

### 4.7 常见陷阱

| 陷阱 | 说明 | 解决方案 |
|:----|:-----|:---------|
| 断点在变化内容上 | 时间戳/随机数导致缓存永不命中 | 把断点放在稳定内容末尾 |
| JSON Key 顺序不稳定 | Swift/Go 可能随机化 Key 顺序 | 确保 JSON 序列化稳定 |
| 并发请求 | 第一个响应开始前缓存不可用 | 等第一个响应后再发后续请求 |
| 修改 tool_choice | 会使缓存失效 | 保持 tool_choice 一致 |

```python
import time

while True:
```

```python
import re

def validate_user_input(user_input: str) -> str:
    """生产级输入验证"""
    # 1. 长度限制
    if len(user_input) > 10000:
        raise ValueError("Input too long")
    
    # 2. 注入攻击检测
    injection_patterns = [
        r"ignore\s+previous\s+instructions",
        r"you\s+are\s+now",
        r"system\s*:\s*",
        r"<\|.*?\|>",
    ]
    for pattern in injection_patterns:
        if re.search(pattern, user_input, re.IGNORECASE):
            raise ValueError("Potential injection detected")
    
    # 3. 敏感信息过滤
    # 移除可能的 API Key、密码等
    user_input = re.sub(r'sk-[a-zA-Z0-9]{20,}', '[REDACTED]', user_input)
    
    return user_input
```

```python
def validate_output(response_text: str, task_type: str) -> dict:
    """验证 Claude 输出的安全性和质量"""
    result = {"safe": True, "issues": []}
    
    # 1. 检查是否泄露系统提示词
    system_prompt_fragments = ["you are an ai", "your instructions are"]
    for frag in system_prompt_fragments:
        if frag in response_text.lower():
            result["safe"] = False
            result["issues"].append("Potential system prompt leakage")
    
    # 2. 检查幻觉标志
    hallucination_markers = [
        "I don't have access to real-time",
        "As an AI language model",
        "I cannot verify",
    ]
    for marker in hallucination_markers:
        if marker in response_text:
            result["issues"].append(f"Potential hallucination: {marker}")
    
    # 3. 结构化输出验证
    if task_type == "json":
        try:
            import json
            json.loads(response_text)
        except json.JSONDecodeError:
            result["safe"] = False
            result["issues"].append("Invalid JSON output")
    
    return result
```

```python
class EscalationManager:
    """管理 AI → 人工升级的决策"""
    
    ESCALATION_TRIGGERS = {
        "high_risk_actions": [
            "delete_account", "process_refund_over_1000",
            "modify_permissions", "access_pii"
        ],
        "confidence_threshold": 0.7,
        "max_retries": 3,
        "sensitive_topics": [
            "legal", "medical", "financial_advice"
        ]
    }
    
    def should_escalate(self, action: str, confidence: float, 
                         topic: str, retry_count: int) -> tuple[bool, str]:
        """判断是否需要升级到人工"""
        
        if action in self.ESCALATION_TRIGGERS["high_risk_actions"]:
            return True, f"High-risk action: {action}"
        
        if confidence < self.ESCALATION_TRIGGERS["confidence_threshold"]:
            return True, f"Low confidence: {confidence:.2f}"
        
        if retry_count >= self.ESCALATION_TRIGGERS["max_retries"]:
            return True, f"Max retries exceeded: {retry_count}"
        
        if topic in self.ESCALATION_TRIGGERS["sensitive_topics"]:
            return True, f"Sensitive topic: {topic}"
        
        return False, "OK"
```

### 7.5 Rate Limiting 与重试策略

```python
import time
from anthropic import RateLimitError, APIError

def robust_api_call(client, **kwargs):
    """带指数退避的健壮 API 调用"""
    max_retries = 5
    base_delay = 1.0
    
    for attempt in range(max_retries):
        try:
            return client.messages.create(**kwargs)
        except RateLimitError as e:
            delay = base_delay * (2 ** attempt)
            print(f"Rate limited. Retry in {delay}s (attempt {attempt + 1}/{max_retries})")
            time.sleep(delay)
        except APIError as e:
            if e.status_code >= 500:  # 服务端错误，可重试
                delay = base_delay * (2 ** attempt)
                time.sleep(delay)
            else:
                raise  # 客户端错误，不重试
    
    raise Exception(f"Failed after {max_retries} retries")
```

### 8.4 成本监控

```python
class CostTracker:
    """跟踪 API 调用成本"""
    
    PRICING = {
        "claude-opus-4-6": {"input": 5.0, "output": 25.0, "cache_read": 0.5, "cache_write": 6.25},
        "claude-sonnet-4-6": {"input": 3.0, "output": 15.0, "cache_read": 0.3, "cache_write": 3.75},
        "claude-haiku-4-5": {"input": 1.0, "output": 5.0, "cache_read": 0.1, "cache_write": 1.25},
    }
    
    def __init__(self):
        self.total_cost = 0.0
        self.calls = []
    
    def track(self, model: str, usage: dict):
        """追踪单次调用成本"""
        pricing = self.PRICING[model]
        
        cost = (
            usage.get("input_tokens", 0) * pricing["input"] / 1_000_000 +
            usage.get("output_tokens", 0) * pricing["output"] / 1_000_000 +
            usage.get("cache_read_input_tokens", 0) * pricing["cache_read"] / 1_000_000 +
            usage.get("cache_creation_input_tokens", 0) * pricing["cache_write"] / 1_000_000
        )
        
        self.total_cost += cost
        self.calls.append({"model": model, "cost": cost, "tokens": usage})
        
        return cost
    
    def report(self):
        print(f"Total cost: ${self.total_cost:.4f}")
        print(f"Total calls: {len(self.calls)}")
        print(f"Avg cost/call: ${self.total_cost / len(self.calls):.4f}")
```
