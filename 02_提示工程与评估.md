### 为什么需要评估？

> 📏 **核心理念**：提示工程不是凭感觉调参数，而是一个**科学的迭代过程**。不评估 = 盲飞。

评估（Evals）帮助你：
- **量化**提示效果（不是"感觉还行"）
- **对比**不同提示版本的优劣
- **防止回归**——修改提示后不会破坏已有功能
- **建立信心**——上生产前确保质量

### 评估类型

| 类型 | 描述 | 适用场景 | 示例 |
|------|------|---------|------|
| **精确匹配** | 输出是否完全匹配预期 | 分类、是/否判断 | 情感分析是否正确 |
| **包含检查** | 输出是否包含关键信息 | 信息提取、摘要 | 是否提到了所有关键事实 |
| **LLM 评审** | 用 Claude 评价 Claude 的输出 | 开放式生成、写作质量 | 邮件是否专业、全面 |
| **人工评审** | 人类专家判断 | 高风险决策、创意写作 | 医疗建议是否准确 |

在这个视频中，我们将走过一个典型提示评估工作流的所有步骤。首先需要理解两件重要的事：第一，组装工作流的方式有很多种，行业中没有一套固定的标准方法；第二，有很多开源包和付费工具可以帮你实现自己的工作流。但在本视频和模块中，我们将在 Jupyter notebook 中从零开始实现自定义工作流，这是为了帮你理解这些工作流的行为，也让你知道不需要重量级解决方案就能做提示评估。

典型提示评估的步骤一：编写初始提示草稿。我们的简单提示只说"请回答用户的问题"，然后插入用户输入。

步骤二：创建评估数据集。数据集包含一些可能输入到提示中的内容。对我们来说，提示只有一个输入——用户问题，所以评估数据集就是一组不同的可能问题。我的数据集只有三个问题，但实际评估中可能有几十、几百甚至几千条记录。数据集可以手动组装，也可以用 Claude 生成。

有了数据集后，我们把每个问题输入提示中，得到完整的提示，然后输入 Claude 获取实际回复。比如第一个问题"2加2等于几"，可能得到回答"2加2等于4"。

然后是评分步骤。我们把数据集中的每个问题和 Claude 的回答配对，逐个输入评分器。评分器会给出一个分数（比如 1-10 分），基于 Claude 回答的质量。10 分意味着完美答案，4 分表示有改进空间。

评分后，我们将所有分数平均。比如得到平均分 7.66，这就给了我们一种客观方式来描述原始提示的表现。然后我们可以修改提示并重新运行整个过程。比如添加更多细节到提示中，如"请详细回答问题"，生成提示 V2，再次运行管道。比较 V1 和 V2 的分数，分数较高的那个版本就可能是更好的提示。

现在我们有了整个过程的高层概述，接下来我们将在 Jupyter notebook 中实现自己的评估框架。

    """辅助函数"""

    prompt = f"""Classify the sentiment of the following text as exactly one of: 
positive, negative, or neutral.

<text>{text}</text>

Respond with only one word: positive, negative, or neutral."""
    return get_completion(prompt).strip().lower()

def run_evaluation(dataset, classifier_fn):
    """运行评估并计算准确率"""
    results = []
    correct = 0
    
    for item in dataset:
        prediction = classifier_fn(item["input"])
        is_correct = prediction == item["expected"]
        if is_correct:
            correct += 1
        results.append({
            "input": item["input"][:50] + "...",
            "expected": item["expected"],
            "predicted": prediction,
            "correct": is_correct
        })

    print()
    
    for r in results:
        status = "✅" if r["correct"] else "❌"
        print(f'{status} Expected: {r["expected"]:8s} | Got: {r["predicted"]:8s} | {r["input"]}')

接下来我们要在提示评估工作流中实现评分系统。评分器会接收模型的输出，然后给出某种客观信号——可能是一个数字或布尔值。非常常见的做法是输出 1-10 的分数，10 表示高质量输出，1 表示低质量输出。

我们要讨论三种评分器：代码评分器、模型评分器和人工评分器。

人工评分器：将所有输出放在真人面前进行评估。人工非常灵活，但缺点是耗时且繁琐。

接下来我们需要实现代码评分器。代码评分器会检查模型输出，确保只返回纯 Python、JSON 或正则表达式，不带任何解释，并验证语法有效性。

### 提示工程的核心思路

> 📏 **Anthropic 提示黄金法则**：把 Claude 当作一个刚加入团队的、聪明但对你的项目一无所知的新同事。你给人类同事的指令能让他们完成任务吗？如果是，那对 Claude 也够了。

提示工程的四大支柱：
1. **清晰指令**：不让 Claude 猜你的意图
2. **结构化数据**：用 XML 标签分离指令和数据
3. **输出格式约束**：指定输出的格式
4. **示例引导**：提供 Few-shot 示例

### 辅助函数（后续各课通用）

```python
import anthropic

CLIENT = anthropic.Anthropic()
MODEL_NAME = "claude-sonnet-4-20250514"

def get_completion(prompt: str, system_prompt: str = "", prefill: str = ""):
    """发送请求并获取 Claude 回复的辅助函数"""
    messages = [{"role": "user", "content": prompt}]
    if prefill:
        messages.append({"role": "assistant", "content": prefill})
    response = CLIENT.messages.create(
        model=MODEL_NAME, max_tokens=2048,
        system=system_prompt, messages=messages
    )
    return response.content[0].text
```

重新运行评估后，分数提升到了 7.96。

总结：one-shot 提供一个示例，multi-shot 提供多个。在处理边界情况或需要匹配复杂输出格式时特别有效。
