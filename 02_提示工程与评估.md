## 第一部分：提示工程交互教程

> 📓 GitHub 仓库：`prompt_engineering_interactive_tutorial/Anthropic 1P/`

### 辅助函数

以下代码是整个教程通用的辅助函数，所有章节的练习都基于此：

```python
import anthropic

# 初始化客户端
CLIENT = anthropic.Anthropic()

# 默认模型
MODEL_NAME = "claude-sonnet-4-20250514"

def get_completion(prompt: str, system_prompt: str = "", prefill: str = ""):
    """
    发送请求并获取 Claude 回复的辅助函数。
    
    参数:
        prompt: 用户消息
        system_prompt: 系统提示（可选）
        prefill: 预填充 assistant 消息开头（可选）
    """
    messages = [{"role": "user", "content": prompt}]
    
    if prefill:
        messages.append({"role": "assistant", "content": prefill})
    
    response = CLIENT.messages.create(
        model=MODEL_NAME,
        max_tokens=2048,
        system=system_prompt,
        messages=messages
    )
    return response.content[0].text
```

### 第 1 章：基本提示结构

> 📓 `01_Basic_Prompt_Structure.ipynb`

#### 核心原则：清晰直接

Claude 就像一个新员工——给它的指令越清晰、具体，效果越好。

**反面示例**：
```python
# ❌ 模糊的提示
response = get_completion("Tell me about dogs.")
# Claude 不知道你想了解哪方面——品种？历史？护理？
```

**正面示例**：
```python
# ✅ 清晰具体的提示
response = get_completion(
    "List the top 5 most popular dog breeds in the US, with a brief description of each breed's temperament."
)
print(response)
```

#### 黄金法则

> 📏 **Anthropic 提示黄金法则**：把 Claude 当作一个刚加入团队的、聪明但对你的项目一无所知的新同事。你给人类同事的指令能让他们完成任务吗？如果是，那对 Claude 也够了。

#### 练习：修复模糊提示

```python
# 原始模糊提示
vague_prompt = "Write something about robots."
print(get_completion(vague_prompt))

# 改进后的具体提示
improved_prompt = """Write a 200-word blog post about the history of industrial robots, 
covering their invention in the 1960s, key milestones, and how they've transformed 
modern manufacturing. Use a professional but accessible tone."""
print(get_completion(improved_prompt))
```

### 第 3 章：使用 XML 标签组织提示

#### 为什么使用 XML 标签？

XML 标签帮助 Claude 清晰区分**指令**和**数据**，特别是当提示包含需要处理的文本时：

```python
# ❌ 不使用标签 — Claude 可能混淆指令和内容
prompt = """Summarize this text: 
The quick brown fox jumps over the lazy dog. 
This is a classic pangram used in typing tests."""

# ✅ 使用 XML 标签 — 清晰分离
prompt = """Please summarize the following text in one sentence.

<text>

This is a classic pangram used in typing tests.
</text>"""

print(get_completion(prompt))
```

### 第 4 章：格式化输出

> 📓 `04_Formatting_Output_and_Speaking_for_Claude.ipynb`

#### 4.1 使用 Prefilling 控制输出格式

**Prefilling（预填充）** 是一个强大技巧：你可以预先填写 assistant 消息的开头，强制 Claude 按特定格式继续生成。

```python
# 强制输出 JSON
response = get_completion(
    "Extract the name, age, and city from: 'John is 30 years old and lives in New York.'",
    prefill='{"name": "'  # 预填充 JSON 开头
)

print('{"name": "' + response)  # 需要手动拼接预填充的部分
```

#### 4.2 强制特定格式

```python
# 强制输出 JSON 格式
prompt = """Analyze the sentiment of the following sentences.
Return ONLY a JSON array where each item has "text" and "sentiment" fields.

Sentences:
1. I love this product!
2. The service was terrible.
3. It's okay, nothing special."""

response = get_completion(prompt, prefill="[")
print("[" + response)
```

#### 4.3 强制 Claude "二选一"

```python
# 使用 prefilling 限制输出为特定选项
prompt = """Is the following statement positive or negative?
Statement: "The movie was a waste of time."
"""

# 预填充让 Claude 只能回答 Positive 或 Negative
response = get_completion(prompt, prefill="The sentiment is: ")
print("The sentiment is: " + response)
```

#### 4.4 输出为 XML 格式

```python
prompt = """Generate a brief character profile for a fantasy RPG character.

Include:
- Name
- Race  
- Class
- Brief backstory (2-3 sentences)

Format your response using XML tags."""

response = get_completion(prompt, prefill="<character>")
print("<character>" + response)
```

#### 为什么需要思维链？

对于需要推理的任务，让 Claude "一步步思考"可以显著提高准确性：

```python
# ✅ 要求思维链 — 更准确
prompt = """A farmer has 17 sheep. All but 9 die. How many sheep are left?

```

```python
Then provide your verdict inside <answer> tags."""
```

### 第 8 章：长文档处理

> 📓 `08_Avoiding_Hallucinations.ipynb`

Claude 支持超长上下文窗口（200K tokens），可以处理整本书、大型代码库等。

#### 长文档处理策略

```python
# 策略 1：文档放在提示开头（减少注意力衰减）
prompt = """<document>
{这里放入你的长文档内容}
</document>

Based on the document above, answer the following questions:
1. What is the main argument?
2. What evidence is presented?
3. What are the conclusions?"""

# 策略 2：分段处理超长文档
def process_long_document(doc_chunks, question):
    """分段处理超长文档"""
    summaries = []
    for i, chunk in enumerate(doc_chunks):
        response = get_completion(
            f"""<chunk number="{i+1}">
{chunk}
</chunk>

Summarize the key points relevant to this question: {question}""",
            system_prompt="You are a document analyst. Extract only relevant information."
        )
        summaries.append(response)
    
    # 汇总所有摘要
    combined = "\n\n".join(summaries)
    final_response = get_completion(
        f"""Based on these summaries from different sections of a document:

<summaries>
{combined}
</summaries>

Provide a comprehensive answer to: {question}"""
    )
    return final_response
```

## 第二部分：提示评估（Prompt Evaluations）

> 📓 GitHub 仓库：`prompt_evaluations/`  
> 📹 对应视频：Lessons on Evaluations

### 为什么需要评估？

> 📏 **核心理念**：提示工程不是凭感觉调参数，而是一个**科学的迭代过程**。你需要量化指标来判断提示的好坏。

评估（Evals）帮助你：
- **量化**提示效果（不是"感觉还行"）
- **对比**不同提示版本的优劣
- **防止回归**——修改提示后不会破坏已有功能
- **建立信心**——上生产前确保质量

### 评估流程总览

```
1. 定义任务 → 2. 收集测试集 → 3. 设计评分标准 → 4. 运行评估 → 5. 分析结果 → 6. 迭代优化
```

### 评估类型

| 类型 | 描述 | 适用场景 | 示例 |
|------|------|---------|------|
| **精确匹配** | 输出是否完全匹配预期 | 分类、是/否判断 | 情感分析是否正确 |
| **包含检查** | 输出是否包含关键信息 | 信息提取、摘要 | 是否提到了所有关键事实 |
| **LLM 评审** | 用 Claude 评价 Claude 的输出 | 开放式生成、写作质量 | 邮件是否专业、全面 |
| **人工评审** | 人类专家判断 | 高风险决策、创意写作 | 医疗建议是否准确 |

```python
def classify_sentiment(text):
    """情感分类 — 被评估的目标提示"""
    prompt = f"""Classify the sentiment of the following text as exactly one of: 
positive, negative, or neutral.

<text>{text}</text>

Respond with only one word: positive, negative, or neutral."""
    
    return get_completion(prompt).strip().lower()
```

#### 步骤 3：运行评估

```python
def run_evaluation(dataset, classifier_fn):
    """运行评估并计算准确率"""
    results = []
    correct = 0
    
    for item in dataset:
        prediction = classifier_fn(item["input"])
        is_correct = prediction == item["expected"]
        
        if is_correct:
            correct += 1
        
        results.append({
            "input": item["input"][:50] + "...",
            "expected": item["expected"],
            "predicted": prediction,
            "correct": is_correct
        })
    
    accuracy = correct / len(dataset) * 100
    
    print(f"总样本数: {len(dataset)}")
    print(f"正确数: {correct}")
    print(f"准确率: {accuracy:.1f}%")
    print()
    
    for r in results:
        status = "✅" if r["correct"] else "❌"
        print(f'{status} Expected: {r["expected"]:8s} | Got: {r["predicted"]:8s} | {r["input"]}')
    
    return accuracy, results

# 运行评估
accuracy, results = run_evaluation(eval_dataset, classify_sentiment)
```

#### 步骤 4：使用 LLM 作为评审（LLM-as-Judge）

对于开放式生成任务，可以用 Claude 评审输出质量：

```python
def llm_judge(question, response, criteria):
    """使用 Claude 作为评审来评判另一个 Claude 的输出"""
    judge_prompt = f"""You are an impartial judge evaluating an AI assistant's response.

<question>
{question}
</question>

<response>
{response}
</response>

<criteria>
{criteria}
</criteria>

Rate the response on a scale of 1-5 for each criterion.
Provide your evaluation in this format:
<evaluation>
<scores>
criterion_name: score (1-5)
</scores>
<overall_score>average score</overall_score>
<explanation>brief explanation of the scores</explanation>
</evaluation>"""
    
    return get_completion(
        judge_prompt,
        system_prompt="You are a strict but fair evaluator. Be specific in your feedback."
    )

# 示例使用
question = "Explain the concept of recursion in programming."
ai_response = get_completion(question)

criteria = """
1. Accuracy: Is the explanation technically correct?
2. Clarity: Is it easy to understand for a beginner?
3. Completeness: Does it cover base case, recursive case, and common pitfalls?
4. Examples: Does it include helpful code examples?
"""

evaluation = llm_judge(question, ai_response, criteria)
print(evaluation)
```

### 2. 多步骤编排（Prompt Chaining）

将复杂任务分解为多个步骤，前一步的输出作为后一步的输入：

```python
def research_and_write(topic):
    """多步骤编排：先研究再写作"""
    
    # 步骤 1：生成大纲
    outline = get_completion(
        f"Create a detailed outline for a blog post about: {topic}",
        system_prompt="You are a content strategist."
    )
    
    # 步骤 2：扩展每个段落
    draft = get_completion(
        f"""Using this outline, write a complete blog post. 
Each section should be 2-3 paragraphs.

<outline>
{outline}
</outline>""",
        system_prompt="You are a skilled technical writer."
    )
    
    # 步骤 3：编辑和优化
    final = get_completion(
        f"""Edit and improve this blog post draft:
        
<draft>
{draft}
</draft>

Focus on:
- Fixing any factual errors
- Improving flow and transitions
- Making technical concepts more accessible
- Adding a compelling introduction and conclusion""",
        system_prompt="You are a senior editor at a major tech publication."
    )
    
    return final
```

## 提示工程技巧速查表

| 技巧 | 使用场景 | 关键点 |
|------|---------|--------|
| **清晰直接** | 所有场景 | 像给新同事写指令一样清晰 |
| **角色设定** | 需要特定风格/专业度 | 使用 system 参数 |
| **XML 标签** | 指令与数据混合 | `<tag>` 分离指令和内容 |
| **Prefilling** | 强制特定输出格式 | 预填充 assistant 消息开头 |
| **思维链** | 推理/计算任务 | 加 "Think step by step" |
| **Few-shot** | 需要精确控制格式/行为 | 提供 2-5 个示例 |
| **防幻觉** | 事实性任务 | 允许说不知道 + 要求引用 |
| **提示链** | 复杂多步骤任务 | 拆解为多个串行调用 |
| **评估管道** | 生产级应用 | 定义测试集 + 自动评估 |
