## 第 1 课：提示评估概论

> 📹 视频：Prompt Evaluations（2 min）  
> 📓 配套 Notebook：`prompt_evaluations/01_evaluations_101.ipynb`

### 为什么需要评估？

> 📏 **核心理念**：提示工程不是凭感觉调参数，而是一个**科学的迭代过程**。不评估 = 盲飞。

评估（Evals）帮助你：
- **量化**提示效果（不是"感觉还行"）
- **对比**不同提示版本的优劣
- **防止回归**——修改提示后不会破坏已有功能
- **建立信心**——上生产前确保质量

### 评估类型

| 类型 | 描述 | 适用场景 | 示例 |
|------|------|---------|------|
| **精确匹配** | 输出是否完全匹配预期 | 分类、是/否判断 | 情感分析是否正确 |
| **包含检查** | 输出是否包含关键信息 | 信息提取、摘要 | 是否提到了所有关键事实 |
| **LLM 评审** | 用 Claude 评价 Claude 的输出 | 开放式生成、写作质量 | 邮件是否专业、全面 |
| **人工评审** | 人类专家判断 | 高风险决策、创意写作 | 医疗建议是否准确 |

## 第 4 课：运行评估

> 📹 视频：Running Evaluations（7 min）  
> 📓 配套 Notebook：`prompt_evaluations/04_classification_eval.ipynb`

### 批量运行评估并计算准确率

```python
import anthropic

CLIENT = anthropic.Anthropic()
MODEL_NAME = "claude-sonnet-4-20250514"

def get_completion(prompt: str, system_prompt: str = "", prefill: str = ""):
    """辅助函数"""
    messages = [{"role": "user", "content": prompt}]
    if prefill:
        messages.append({"role": "assistant", "content": prefill})
    response = CLIENT.messages.create(
        model=MODEL_NAME, max_tokens=2048,
        system=system_prompt, messages=messages
    )
    return response.content[0].text

def classify_sentiment(text):
    """情感分类 — 被评估的目标提示"""
    prompt = f"""Classify the sentiment of the following text as exactly one of: 
positive, negative, or neutral.

<text>{text}</text>

Respond with only one word: positive, negative, or neutral."""
    return get_completion(prompt).strip().lower()

def run_evaluation(dataset, classifier_fn):
    """运行评估并计算准确率"""
    results = []
    correct = 0
    
    for item in dataset:
        prediction = classifier_fn(item["input"])
        is_correct = prediction == item["expected"]
        if is_correct:
            correct += 1
        results.append({
            "input": item["input"][:50] + "...",
            "expected": item["expected"],
            "predicted": prediction,
            "correct": is_correct
        })
    
    accuracy = correct / len(dataset) * 100
    
    print(f"总样本数: {len(dataset)}")
    print(f"正确数: {correct}")
    print(f"准确率: {accuracy:.1f}%")
    print()
    
    for r in results:
        status = "✅" if r["correct"] else "❌"
        print(f'{status} Expected: {r["expected"]:8s} | Got: {r["predicted"]:8s} | {r["input"]}')
    
    return accuracy, results

# 运行评估
# accuracy, results = run_evaluation(eval_dataset, classify_sentiment)
```

```python
def llm_judge(question, response, criteria):
    """使用 Claude 作为评审来评判另一个 Claude 的输出"""
    judge_prompt = f"""You are an impartial judge evaluating an AI assistant's response.

<question>
{question}
</question>

<response>
{response}
</response>

<criteria>
{criteria}
</criteria>

Rate the response on a scale of 1-5 for each criterion.
Provide your evaluation in this format:
<evaluation>
<scores>
criterion_name: score (1-5)
</scores>
<overall_score>average score</overall_score>
<explanation>brief explanation of the scores</explanation>
</evaluation>"""
    
    return get_completion(
        judge_prompt,
        system_prompt="You are a strict but fair evaluator. Be specific in your feedback."
    )

# 示例使用

4. Examples: Does it include helpful code examples?
```

## 第 8 课：提示工程总论

> 📹 视频：Prompt Engineering（11 min）  
> 📓 配套 Notebook：`prompt_engineering_interactive_tutorial/` 全系列

### 提示工程的核心思路

> 📏 **Anthropic 提示黄金法则**：把 Claude 当作一个刚加入团队的、聪明但对你的项目一无所知的新同事。你给人类同事的指令能让他们完成任务吗？如果是，那对 Claude 也够了。

提示工程的四大支柱：
1. **清晰指令**：不让 Claude 猜你的意图
2. **结构化数据**：用 XML 标签分离指令和数据
3. **输出格式约束**：指定输出的格式
4. **示例引导**：提供 Few-shot 示例

### 辅助函数（后续各课通用）

```python
import anthropic

CLIENT = anthropic.Anthropic()
MODEL_NAME = "claude-sonnet-4-20250514"

def get_completion(prompt: str, system_prompt: str = "", prefill: str = ""):
    """发送请求并获取 Claude 回复的辅助函数"""

    if prefill:
        messages.append({"role": "assistant", "content": prefill})
    response = CLIENT.messages.create(
        model=MODEL_NAME, max_tokens=2048,

```

> 📹 视频：Be Specific（5 min）

### 技巧二：越具体越好

给出约束（长度/格式/语气/受众），而不是模糊的"make it good"：

```python
# ❌ 模糊
"Translate this to French and make it sound good."

# ✅ 具体
"Translate the following product description into French. 
Use a formal tone suitable for a luxury brand website. 
Keep the translation under 150 words."
```

### 格式化输出与 Prefilling

**Prefilling（预填充）** 是一个强大技巧：预先填写 assistant 消息的开头，强制 Claude 按特定格式继续生成。

```python
# 强制输出 JSON
response = get_completion(
    "Extract the name, age, and city from: 'John is 30 years old and lives in New York.'",
    prefill='{"name": "'
)
print('{"name": "' + response)
```

> 📹 视频：Structure with XML Tags（4 min）

### 技巧三：用 XML 标签分离指令和数据

XML 标签帮助 Claude 清晰区分**指令**和**数据**，特别是当提示包含需要处理的文本时：

```python
# ❌ 不使用标签 — Claude 可能混淆指令和内容
prompt = "Summarize this text: The quick brown fox jumps over the lazy dog."

# ✅ 使用 XML 标签 — 清晰分离
prompt = """Please summarize the following text in one sentence.

<text>
The quick brown fox jumps over the lazy dog. 
This is a classic pangram used in typing tests.
</text>"""
```

### 多步骤任务中的标签使用

```python
prompt = """I need you to perform two tasks on the text below.

Machine learning is a subset of artificial intelligence that focuses on building 
systems that learn from data. Unlike traditional programming where rules are 
explicitly coded, ML systems identify patterns and make decisions with minimal 
human intervention.
</text>

<task1>
Summarize the text in exactly 2 sentences for a technical audience.
</task1>

<task2>
Explain the same concepts using an analogy that a 10-year-old would understand.
</task2>
```

### 思维链推理（Chain of Thought）

对于需要推理的任务，让 Claude "一步步思考"可以显著提高准确性：

```python
prompt = """A farmer has 17 sheep. All but 9 die. How many sheep are left?

Think through this step by step before giving your answer."""
```

使用 `<thinking>` 标签分离推理过程和最终答案：

```python
prompt = """Determine if the following argument is logically valid.

Argument:
"All cats are mammals. Some mammals can fly. Therefore, some cats can fly."

First, analyze the logical structure inside <thinking> tags.
Then provide your verdict inside <answer> tags."""
```

## 提示工程技巧速查表

| 技巧 | 使用场景 | 关键点 |
|------|---------|--------|
| **清晰直接** | 所有场景 | 像给新同事写指令一样清晰 |
| **角色设定** | 需要特定风格/专业度 | 使用 system 参数 |
| **XML 标签** | 指令与数据混合 | `<tag>` 分离指令和内容 |
| **Prefilling** | 强制特定输出格式 | 预填充 assistant 消息开头 |
| **思维链** | 推理/计算任务 | 加 "Think step by step" |
